How to use arguments

Use it by writing ? after API link and then =, use ampersant (&) to set more then one argument.

Example

POST /api/solar_system/30004147/signature?name=5&type=2

return signatures in solar system with id=30004147 and signature should be type=2 and name=5 (type and name is id in database for signature type and name)

Regions

Arguments:

  • id - list of ids of regions you want to get (comma is delimiter)
  • name - list of names of regions you want to get (comma is delimiter)

GET /api/region

return list of all exists regions in database (arguments is filters of regions)

GET /api/region/int:region_id

return information about region. (arguments is filters)

POST /api/region/int:region_id

create new region. Need arguments name.

Solar systems:

Arguments:

  • id - list of ids of solar systems you want to get (used only for /api/solar_system; comma is delimiter)
  • name - name of solar system you want to get
  • region - region id of solar system

GET /api/solar_system

return list of all exists solar systems in database (arguments is filters of solar systems; argument name can be list as id)

GET /api/solar_system/int:solar_system_id

return information about <solar_system_id> (arguments is filters)

POST /api/solar_system/int:solar_system_id

create new solar system with id=<solar_system_id> and other required information from arguments (need all of them)

Signatures:

Arguments:

  • code - list of signatures codes (comma is delimiter)
  • name - names id of signature (always single value)
  • type - types id of signature (always single value)

GET /api/solar_system/int:solar_system_id/signature

return list of all known signatures for solar system with id=<solar_system_id> (arguments is filters)

POST /api/solar_system/int:solar_system_id/signature

create new signatures in solar system with id=<solar_system_id> (Arguments is addition information. Always require codes. If codes list has more then one signature, all signatures will create with same type and name)

PUT /api/solar_system/int:solar_system_id/signature

update signature in solar system with id=<solar_system_id>(Arguments is addition information. Always require codes. Can update only one signature per request)

Signature names:

Arguments:

  • id - id of signature name (comma is delimiter)
  • name - english name of signature name (comma is delimiter)

GET /api/signature_name

return list of all known signature names (arguments is filters)

POST /api/signature_name

create new signature name (All arguments is needed. Can create only one signature name per request)

PUT /api/signature_name

update signature name with id (All arguments is needed. Can update only one signature name per request)

Signature types:

Arguments:

  • id - id of signature type(comma is delimiter)
  • name - english name of signature type (comma is delimiter)

GET /api/signature_name

return list of all known signature types (arguments is filters)
